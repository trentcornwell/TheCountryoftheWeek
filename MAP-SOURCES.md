# Country Map Sources

Documents where the country outline maps at
[`theme/country-week/assets/maps/countries/`](theme/country-week/assets/maps/countries/)
come from, how they were built, and which countries needed manual/special
handling. Generated by [`scripts/build-country-maps.py`](scripts/build-country-maps.py) —
see that script's docstring for the full pipeline.

## Source dataset

- **Dataset**: Natural Earth, 1:10m Cultural Vectors, "Admin 0 – Countries"
- **Source URL**: https://raw.githubusercontent.com/nvkelso/natural-earth-vector/master/geojson/ne_10m_admin_0_countries.geojson
  (a ready-to-use GeoJSON mirror of the official Natural Earth shapefile,
  maintained by a Natural Earth core author, used here since it avoids a
  separate shapefile-to-GeoJSON conversion step)
- **License**: Public domain. Natural Earth's own terms state "You may use
  the maps in any manner, including modifying the content and design,
  electronic dissemination, and printed publications... No permission is
  needed to use Natural Earth."
- **Attribution requirement**: None required by the license. This document
  itself serves as the project's internal record of provenance.
- **Date downloaded**: 2026-07-18
- **File cached at**: `.cache/natural-earth/ne_10m_admin_0_countries.geojson`
  (13,287,234 bytes; git-ignored — re-downloaded on demand by the build
  script, see "Regenerating" below)

## Country list

This project does **not** maintain a second country list. Every map is
generated for exactly the countries in `data/country-index.json` (the
site's single canonical, authoritative manifest), matched onto Natural
Earth features by ISO 3166-1 alpha-2 code (reusing the `iso_codes` mapping
already present in `data/factbook-media-codes.json` for the site's flag
images). The resulting per-country mapping — manifest key, ISO alpha-2,
ISO alpha-3, matched Natural Earth feature name, and output filename — is
written to [`data/country-map-index.json`](data/country-map-index.json).
That file is fully regenerated on every build; never hand-edit it.

Four countries (France, Norway, Kosovo, Taiwan) have `ISO_A2 = "-99"` in
Natural Earth for political-sensitivity reasons; the build script falls
back to Natural Earth's `ISO_A2_EH` field for those.

## Modifications performed

Per country, geometry goes through: flatten (MultiPolygon → polygons) →
antimeridian unwrap (if needed) → invalid-geometry repair (if needed) →
Douglas-Peucker simplification (0.02°, ~1-2km at the equator) →
geographic clustering → primary-cluster selection → local equirectangular
projection → scale/center into the 1000×1000 canvas (800-unit content,
100-unit padding per side) → minimum-visible-feature-size enforcement →
canvas-space simplification (1.5 units) → SVG serialization. Full detail
on each non-trivial step below.

### Countries requiring special manual handling

**1. Antimeridian unwrapping.** Fiji, Kiribati, New Zealand, Russia, and
the United States each have territory that crosses the 180° line. Plotted
naively, longitudes jump between +179° and -179°, producing a shape torn
in half across the canvas. Fixed by detecting a >180° longitude span and
shifting negative longitudes by +360° before projecting, so the country's
own geometry stays contiguous.

**2. Distant islands excluded from framing.** Kiribati, France, Mauritius,
Netherlands, Norway, and Portugal each have island territory far enough
from their main landmass that including it in the bounding box used for
scaling would shrink the main landmass to an unreadable speck (this was
the original bug this project's validation caught for Kiribati — see
below). Nearby islands are always geographically clustered together first
(single-linkage clustering on bounding-box gap, `CLUSTER_GAP_DEGREES =
4.0`), so an archipelago nation is never reduced to a single island. Only
whole *clusters* that are both distant and small are dropped — specifically
when including them would expand the framing bounding box beyond
`PRIMARY_CLUSTER_MAX_EXPANSION = 3.0` times the main cluster's own size —
and dropped clusters are omitted from the drawing entirely, not rendered
off-canvas. Exact per-country counts are in
`reports/country-map-build-report.md`.

  - **Kiribati** (explicitly required by spec): Gilbert Islands, Phoenix
    Islands, and Line Islands are separated by roughly a third of the
    globe. The Gilbert Islands (largest population, the capital Tarawa)
    anchor the frame; 32 more-distant island polygons are excluded from
    the bounding box calculation and not drawn. This is a deliberate,
    geographically honest tradeoff for a nation whose defining
    characteristic *is* being extremely spread out — the alternative
    (including everything) would render Kiribati as an unreadable speck,
    which the task spec explicitly says to avoid.

**3. Source geometry repaired.** Egypt's Natural Earth mainland ring
self-intersects at one vertex (a data quality issue in the source, not
something introduced by this pipeline) and Shapely flags it `is_valid ==
False`. The first version of this script *filtered out* invalid polygons
rather than repairing them — which silently discarded 100% of Egypt's
land area, leaving only a handful of tiny Red Sea/Nile Delta island
fragments as the entire "map of Egypt." Caught by direct visual
inspection of the output (not just trusting the "196/196 succeeded"
summary line — the build reports success even though the shape was
wrong, since nothing about it was structurally invalid SVG). Fixed by
repairing invalid polygons with the standard `buffer(0)` trick
(`repair_polygon()` in the build script) instead of dropping them.

**4. Features grown to a minimum visible size.** Marshall Islands,
Micronesia, Maldives, Tuvalu, and Seychelles are composed entirely of
small atolls/islets spread across a disproportionately large area. Scaled
proportionally to fill the canvas the way every other country is, their
largest single island was only ~6–16 canvas units across (out of 1000) —
correct geometry, but rendering as a near-invisible speck or nothing at
all, which is a real cartographic failure mode for tiny island nations,
not something the antimeridian/clustering logic above addresses (those
countries only exclude *distant* clusters; these nations don't have a
"main population center" cluster big enough to exclude anything relative
to). Fixed by growing (via a symmetric buffer, centered on the feature's
own position — nothing is moved) any single polygon smaller than
`MIN_FEATURE_SIZE = 14` canvas units, after projection/scaling, up to
that minimum. This is the standard cartographic minimum-symbol-size
technique used when a real feature's true-scale size would be
imperceptible. It also incidentally affects many other countries with
small offshore islands (see `reports/country-map-build-report.md` for
the full per-country list and counts) — always harmless, since it never
touches anything already at or above the minimum size, so a country's
main landmass and any normal-sized island are never affected.

### File-size note

The minimum-feature-size buffer initially produced very high-resolution
circles (Shapely's default buffer resolution), ballooning total output
from ~1.5MB to ~4.2MB — worst case Canada at 583KB for one file, from
hundreds of tiny Arctic islands each becoming a 32+-point circle. Fixed
by using a coarser buffer approximation (`quad_segs=2`, an octagon
instead of a 32-gon — visually indistinguishable at a 14-unit feature
size) and adding a final canvas-space simplification pass (1.5 units,
well under one display pixel at any realistic size) applied to every
polygon before serialization. Final total: ~985KB across all 196 files
(~5KB average) — smaller than the pre-fix baseline despite fixing three
real bugs.

## Regenerating

```
python scripts/build-country-maps.py
python scripts/build-country-maps.py --force-download   # re-fetch Natural Earth data
```

Safe to rerun at any time: the Natural Earth download is cached and only
refetched if missing (or `--force-download` is passed); every output
file — SVGs, `data/country-map-index.json`,
`reports/country-map-build-report.md` — is fully regenerated from source
on each run, so there's no partial/incremental state to get out of sync.

## Validation performed

See `reports/country-map-build-report.md` for the automated pass
(well-formed XML, exact `viewBox="0 0 1000 1000"`, no `<script>`/`on*=`
content, per-country special-handling notes) and file-size totals. Beyond
the automated checks, every generated SVG was rasterized with a small
custom preview script and visually inspected for correct proportions,
centering, and absence of stretching — specifically including the nine
countries called out in the task spec (Kiribati, Chile, Russia, Indonesia,
United States, United Kingdom, Philippines, Fiji, plus Egypt after its
bug was found) and a broader spot-check of other multi-island/archipelago
nations (Japan, New Zealand, Greece, Solomon Islands, Marshall Islands,
Seychelles, Maldives, Comoros, Vanuatu, Bahamas, Cabo Verde, Micronesia,
Tuvalu, Tonga, Palau, Canada).
